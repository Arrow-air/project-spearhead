# Aerodynamic Sizing Toolchain

Initial aerodynamic sizing for the Spearhead UAV. Determines wing area, tail incidence, cruise drag, and stall behavior for any airfoil combination from a 1,634-airfoil database.

## Quick Start

All inputs are in **`config.py`**. Edit the parameters, set the mode, and run:

```bash
python config.py
```

### Modes

| MODE | What it does |
|------|-------------|
| 1 | **Airfoil sweep** — tests all airfoil combinations from the database, ranks by composite score, outputs top 30 |
| 2 | **Specific sizing** — runs the sizing tool for the airfoils specified in the `AIRFOIL` dict, prints full results and L/D plots |

## File Structure

| File | Purpose |
|------|---------|
| `config.py` | All inputs — aircraft parameters, airfoil selection, sweep filters, scoring weights. Single entry point. |
| `aircraft_sizing.py` | Core sizing solver — atmosphere, wing sizing, trim, drag, stall, alpha sweep |
| `airfoil_sweep.py` | Two-stage airfoil sweep — wing candidates then wing+tail combinations |
| `airfoil_db_re500000.csv` | Airfoil database at Re=500k (1,634 airfoils) |
| `airfoil_db_re1000000.csv` | Airfoil database at Re=1M (1,634 airfoils) |
| `oratex_compatibility.csv` | Airfoil concavity data for Oratex skin filter (generated by `download_geometries.py`) |

## config.py Structure

```
MODE        — 1 (sweep) or 2 (specific sizing)

AIRCRAFT    — shared by both modes
  Atmosphere, flight conditions, MTOW
  Fuselage geometry
  Wing geometry (AR, incidence, CG position)
  Tail configuration (1=V-tail, 2=conventional)
  Tail geometry (span, chord, dihedral, CG position)
  Vertical fin (config 2 only)
  Drag settings (misc factor, base drag recovery)
  Propeller thrust moment
  Tail incidence override

AIRFOIL     — used by MODE 2 (specific sizing)
  Airfoil input mode (1=DB lookup, 2=manual coefficients)
  Wing and tail airfoil names or manual aero coefficients
  Reynolds number

SWEEP       — used by MODE 1 (airfoil sweep)
  Oratex skin filter
  Excluded airfoils list
  Wing filters (cl_max, cd_0, thickness)
  Tail filters (camber, cd_0, thickness)
  Stage control (top wing candidates, skip tail)
  Scoring weights (area, drag, tail incidence, stall)
  Output options (top N, detail rank)
```

## Tail Configuration

### Config 1: V-tail / Inverse V-tail
- `tail_proj_span` = horizontal projected span
- `tail_dihedral` = panel dihedral angle (rad)
- Actual span = projected / cos(dihedral)
- Tail provides both pitch (V_h) and yaw (V_v) authority

### Config 2: Conventional tail
- `tail_proj_span` = full horizontal tail span
- `tail_dihedral` is ignored (set to 0 internally)
- `vert_chord` and `vert_span` define the separate vertical fin for V_v

## Key Equations

- **Wing area**: `S = W / (q_stall * CL_max_3d)`
- **3D lift slope** (Helmbold): `CL_alpha = a0 / (sqrt(1 + k^2) + k)` where `k = a0 / (pi * e * AR)`
- **3D cl_0 correction**: `cl_0_3d = cl_0 * (CL_alpha_3D / cl_alpha_2d)` — preserves zero-lift angle
- **Oswald efficiency** (Raymer): `e = 1.78 * (1 - 0.045 * AR^0.68) - 0.64`
- **Induced drag**: `CDi = CL^2 / (pi * AR * e)`
- **Fuselage drag**: Prandtl-Schlichting friction + Raymer form factor + Hoerner 3D base drag
- **Base drag recovery**: `D_base = D_base_full * (1 - recovery)`, default 34% recovery for pusher prop
- **Downwash**: `epsilon = factor * 2 * CL_wing / (pi * AR)`
- **V_h** (effective): `S_tail * cos(dihedral) * l_tail / (S_wing * c_wing)`
- **V_v**: `S_tail * sin(dihedral) * l_tail / (S_wing * b_wing)` (V-tail) or `S_vert * l_tail / (S_wing * b_wing)` (conventional)

## Sweep Scoring

Candidates are ranked by a weighted composite score (lower = better):

```
Score = w_area * norm(S_wing) + w_drag * norm(D_total) + w_itail * norm(|i_tail|) + w_stall * norm(M_stall_penalty)
```

Default weights (configurable in `config.py`):
- 40% wing area (smaller = lighter)
- 40% total drag (lower = more efficient)
- 10% tail incidence (closer to zero = less trim drag)
- 10% stall safety (nose-down moment preferred)

## Oratex Skin Filter

The Oratex filter excludes airfoils with concave lower surfaces (incompatible with heat-shrink skin). It requires:

1. Airfoil geometry files (`.dat`) in `../geometry/`
2. `oratex_compatibility.csv` — generated by running `download_geometries.py` in the parent `Sizing/` folder

If `oratex_compatibility.csv` is missing, the filter is automatically disabled with a warning. The sweep will still run but won't exclude concave airfoils.

## CFD Validation

The sizing tool has been validated against RANS CFD and AVL vortex-lattice:
- Lift: within 5-6% of CFD across 0-13 deg AoA
- Drag: within 6% of CFD with the base drag recovery model
- Propeller interaction (actuator disk): confirms base drag recovery assumption

See `aero_validation_note.md` in the parent directory for full details.
